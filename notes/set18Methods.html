Skill 18.2 Write code to create a method
Skill 18.2 Concepts

In our previous example, every time the user wanted to make a deposit, make a withdrawel, or check their balance we had to add code to the main method of our program. Can you imagine how messy it would look if you needed to make 10 deposits, followed by 10 more balance inquiries?

Methods in java can be used to define such behaviors and can be applied to greatly simplify our code. Consider checkBalance method below.

public void checkBalance{

System.out.println("Your balance is " + balance);

}

The first line, public void checkBalance(), is the method signature. It gives the program some information about the method. each part of this line is described below,

  • public means that other classes can access this method. We will learn more about that later.
  • The void keyword means that there is no specific output from the method. We will see methods that are not void later, but for now all of our methods will be void.
  • checkBalance is the name of the method.

The statement System.out.println("Your balance is " + balance); is in the body of the method, which is defined by the curly braces. Anything we can do in our main() method, we can do in other methods! All of the Java tools you know, like the math and comparison operators, can be combined to create useful methods.

Now that we have added the checkBalance() method to our SavingsAccount class, it is available to use on any SavingsAccount object. We can do this by calling the method on the SavingsCount object we create, for example.

SavingsAccountSavingsAccountDriver

public class SavingsAccount{

public String name;
public double balance;

public SavingsAccount(String n, double b){

name = n
balance = b
}
public void checkBalance(){

System.out.println("Your balance is $" + balance);
}
}

public class SavingsAccountDriver{

public static void main(String args[]){

SavingsAccount account1 = new SavingsAccount("Bart", 125.00);
account1.checkBalance();
}
}

In the example above, we call the checkBalance() method on the account1 object. This method call occurs inside the main() method. Running the program results in printing Your balance is $125.00 to the console.

Skill 17.3 Instantiate a class
Skill 17.3 Concepts

Instantiating is a fancy word for creating. If you instantiate a class you are creating a data type of that class. In a previous example we defined a Student data type called student1 and another Student data type called student2. We did this by instantiating the Student class.

To instantiate a class in java we use the new key word as illustrated below.

public class StudentMaker{

public static void main(String args[]){

Student student1 = new Student();
}
}

The code above created an instance of the Student class called student1. student1 is also referred to as a Student object

Notice, that we created our Student object, student1 in the main method of a different class called StudentMaker. We were able to do this because the student class was given the public access modifier. Below are the two classes side by side,

Skill 17.4 Pass parameters to a constructor
Skill 17.4 Concepts

A parameter is a value you can pass to a method (or a constructor). For example, rather than defining the students name, gradeLevel, GPA, etc. after the Student object has been created, this can be done all at once. Before we can do so however, we must modify the constructor in the Student class to allow for parameters.

public class Student{

public String name;
public int gradeLevel;
public double GPA;
public boolean hasScholarship;

public Student( String n, int g, double gpa, boolean s ){

}
}

Once the values are received by the constructor, they can be assigned to the variables name, gradeLevel, GPA, and hasScholarship as follows,

public class Student{

public String name;
public int gradeLevel;
public double GPA;
public boolean hasScholarship;

public Student( String n, int gl, double gpa, boolean s ){

name = n;
gradeLevel = gl;
GPA = gpa;
hasScholarship = s;
}
}

Now, that we have modified the constructor, we must also modify our code to create our Student instances. Below, we create two Student instances, student1 and student2,

public class StudentMaker{

public static void main(String args[]){

Student student1 = new Student("Bart", 10, 2.6, false);
Student student2 = new Student("Kyle", 11, 3.5, true);
}
}
